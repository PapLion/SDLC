# Roadmap.sh

## Software Design & Architecture

### Clean code

Be consistent,Meaningful Names, Identation and code style, keep methods/classes/files small, pure functions, minimize cyclomatic complexity, avoid passing nulls, booleans, keep framework code distant, use correct cosntructs, teats sould be fast and independent, organize code by actor it belongs to, command query separation, keep it simple and refactor often

### Programing paradigms

Structured, functional, oop(PARADIGMS, MDD)

### Design principles

Solid, dry, yagni, low demeter, tell dont ask, holywood principle, composition over inheritance, encapsulate what varies, program agaist abstractions

### Design patterns

Gof design patterns, posa patterns

### Architectural principles

Component principles, policy vs detail, coupling and cohesion, boundaries

### Architectura Styles

- Structural: monolithic, layered, component-based
- Messaging: Event Driven, publish subscribe
-Distributed: Client-Server, Peer-to-peer

### Architectural patterns

Domain-Driven Design, Model-View Controller, Microservices, Blockboard pattern, MicroKernel, services architecture, message queues/streams, event sourcing, soa, cqrs

### Enterprise patterns

DTOs, Identity maps, usecases, repositories, mappers, transaction script, commands/queries, value objects, domain models, entities, qrms
